# AI-Native macOS 审计引擎（用户快速上手）

这是一款用于审计和优化你 Mac 应用环境的工具。  
你可以用它查看应用来源、权限风险、更新建议，以及通过 AI 做语义检索。

## 1) 安装

当前版本为开发者预览版，建议由开发者帮你本地运行：

```bash
swift build
swift run
```

启动后会打开应用主界面。

## 2) 第一次使用（3 步）

1. 打开应用，等待扫描完成（会显示“已加载 X 个应用”）。
2. 在“语义查询”里填入：
  - API Key
  - Base URL
  - Model
3. 输入问题并点击“运行查询”。

示例问题：

- `帮我找出所有带后台常驻进程的软件`
- `哪些软件最近占用了大量 CPU 资源`
- `列出第三方来源且风险较高的软件`

## 3) 你会看到什么

### 风险审计

- 展示可疑应用和风险原因（例如：未签名、权限异常、后台常驻+高敏权限）。

### 语义查询

- 你可以像聊天一样查找应用集合，AI 会返回解释和建议。

### 运维建议

- 展示更新建议（优先 Homebrew Cask 匹配）。
- 展示当前版本能力状态（DIRECT/MAS）。

## 4) DIRECT 与 MAS 的区别

- `DIRECT`：能力完整（默认）。
- `MAS`：部分敏感能力受限（例如某些系统级读取能力）。

如果你看到某些能力显示“受限”，通常是构建策略导致，不是应用故障。

## 5) 常见问题

### AI 无法返回结果

- 检查 API Key 是否正确。
- 检查 Base URL 是否可访问、是否兼容 OpenAI 格式接口。
- 检查 Model 名称是否可用。

### 权限状态显示 unavailable

- 当前构建可能为受限模式（如 MAS）。
- 系统权限策略限制了读取能力。

### 更新建议为空

- 你的机器可能没有安装 Homebrew，或应用未匹配到 cask。

## 6) 安全与隐私说明（简版）

- 工具会尽量只发送最小化的应用摘要信息给 AI。
- 不会把完整应用文件上传到模型服务。
- 不同构建模式下，能力范围会不同。
